# Sql---data-warehouse-project
Building a modern data warehouse with SQL server , including ETL processes, data modelling and analytics 
A Data Warehouse is a centralized system used to store large volumes of structured data from multiple sources. It is designed specifically for reporting, analysis, and decision-making, rather than daily transaction processing.

A Data Warehouse Project in SQL involves designing, building, and querying this system using SQL to transform raw data into meaningful insights.

## Project Requirement
### Building the data warehouse (data -engineering)

### objective
Combine data from different sources (ERP, CRM, Excel, APIs)
Clean and transform data
Store historical data for analysis
Enable fast querying for reports and dashboards


### Specifcation 
**Data Source** :Import data two source system (ERP and CRM) providede as csv files.
**Data Quality** cleanse and resolve data quality issues prior to analysis.
**Integration** Combining data from multiple sources into a unified, consistent format in the data warehouse.
**Scope** Defines the boundaries, objectives, and data coverage included in the data warehouse project.
**Documentation** Detailed records of data models, ETL processes, and system design for reference and maintenance.

### Objective
Develop SQL-based ananlytics to deliver detailed insights into:
**Customer Behaviour**
**Product Performance**
**Sales Trend**
